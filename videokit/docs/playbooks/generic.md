# Generic Playbook

## 向いている題材

- まだ型が決まっていない企画
- 解説でも実況でもない中間案件
- テンプレート選定前の叩き台

## 企画の切り方

- まず `誰に何を見せたいか` を決める
- その後で `explainer / gameplay / line-chat / political` のどれに近いか判定する
- 迷う場合は explainer ベースで始める

## Script の作り方

- まず最小限の `hook -> body -> takeaway`
- その後、適した template に寄せて schema を具体化する

## レイアウトの考え方

- 派手さより可読性
- subtitle と popup の競合を避ける
- 背景は意味のあるものだけ使う

## 字幕 / popup / 背景の使い分け

- 字幕: コアメッセージ
- popup: 構造化、強調
- 背景: 文脈補助

## 音声 / BGM / SE

- まず静かめに始める
- 役割が見えてから sceneRole に沿って足す

## 素材収集

- 公式 source か rights-aware source を使う
- 雰囲気だけの素材を集めすぎない

## Render 前 review

- `review:preflight`
- 必要なら `review:layout`

## よくある失敗

- generic のまま最後まで走る
- 途中で template を確定せず schema がぶれる
